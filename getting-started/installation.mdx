## Requirements

| Requirement | Minimum version | How to check |
|---|---|---|
| OS | Linux, macOS, Windows (WSL) | — |

<Note>
  On Windows, RemoteAgent.CHAT is supported and tested inside **WSL 2** (Windows
  Subsystem for Linux). Native Windows support via PowerShell or CMD is not
  guaranteed.
</Note>

## Install RemoteAgent.CHAT

Download and install the binary with a single command:

```bash
curl -fsSL https://remoteagent.chat/install | bash
```

### Verify the installation

```bash
remoteagent --version
```

Expected output:

```
remoteagent x.y.z
```

## Runner prerequisites

RemoteAgent.CHAT itself has no additional dependencies. However, the AI runner you choose may require its own tooling installed separately.

### claude-code (default)

Requires the `claude` CLI binary:

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

Authenticate with a Claude Pro or Max subscription (no API key needed) or with an Anthropic API key. See the [claude-code runner docs](/runners/claude-code) for all authentication options.

### claude-sdk

No additional binary needed. The runner uses the `@anthropic-ai/claude-code` SDK in-process. You will need an **Anthropic API key**, which you provide during `remoteagent init` and is stored locally in `~/.remoteagent/agents/{agentId}.json`. Each project can use a different key.

### gemini

Requires the Google Gemini CLI:

```bash
npm install -g @google/gemini-cli
gemini --version
```

A Google account is sufficient for the free tier. No credit card required.

### opencode

Requires the OpenCode CLI:

```bash
npm install -g opencode-ai
opencode --version
```

No API key required to get started — OpenCode supports 75+ providers including GitHub Copilot and local models via Ollama.

### aider

Requires Python 3.9+ and pip:

```bash
pip install aider-chat
aider --version
```

Aider works with OpenAI, Anthropic, Ollama, and many other model providers. You configure the model and API key separately via aider's own config or environment variables.

### openclaw

Requires the OpenClaw gateway to be running locally on port `18789` before starting the agent. Refer to the [OpenClaw runner documentation](/runners/openclaw) for setup details.

### custom

No specific prerequisites — any binary that reads from stdin and writes to stdout will work. See [Custom runner](/runners/custom) for examples.

## Updating

Use the built-in reinstall command — it updates the binary and automatically restarts any running agents:

```bash
remoteagent reinstall
```

## Uninstalling

Remove the binary and optionally the local configuration:

```bash
rm $(which remoteagent)
# Optional: remove local config
rm -rf ~/.remoteagent
```
